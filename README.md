## Mission Control

I'm a Year 4 Mechanical Engineering undergrad (BEng, Honours) at NTU, building the perception side of autonomous vehicles with **Archimedes Autonomous Vehicles (AAV)**. My focus is turning what a camera sees into where a boat goes, from detection and visual servoing in ROS 2 to the web tools the team runs on.

```txt
Paperdragone@vrx-sim:~$ ./launch_profile --mode perception-autonomy

STATUS        : mechanical engineering undergrad (BEng Honours), NTU
CURRENT       : image-guided planning and control of a WAM-V USV in VRX
SPECIALTY     : computer vision and perception - detection, visual servoing, camera-to-control
OPERATING MODE: test in sim, then close the loop from a single camera
```

| Console | Signal |
|---|---|
| **Perception** | OpenCV, YOLO (Ultralytics), single-camera gate detection |
| **Autonomy** | ROS 2, yaw/surge control, visual servoing |
| **Simulation** | VRX, Gazebo, WAM-V USV |
| **Web and ops** | Next.js, React, TypeScript, Tailwind, Supabase, Vercel |
| **Competitions** | RobotX, SUAS |

## What I'm Building

Most team work lives in Archimedes' private repositories, so the public graph is only part of the story.

| Project | Stack | What it does |
|---|---|---|
| **Image-Guided Planning and Control of a USV** | ROS 2, OpenCV, YOLO, VRX / Gazebo | Steers a WAM-V unmanned surface vehicle through buoy gates from a single camera (YOLO to gate measurement to yaw/surge control) |
| **AAV Website** | Next.js, React, TypeScript, Supabase, Vercel | The team's public website ([`aav-website-2026`](https://github.com/Paperdragone/aav-website-2026)) |
| **AAV Ops Portal** | Next.js, React, Tailwind | The team's internal operations dashboard |

## Field Notes

| Track | Focus | Why it matters |
|---|---|---|
| **Perception** | Clean detections before clever control | You can't steer on noisy measurements |
| **Autonomy** | Simple ROS 2 interfaces, camera to control | Fewer moving parts survive integration |
| **Simulation** | VRX parity before the water | Catch it in sim, not on race day |
| **Web and ops** | Ship the team's tools and keep them current | The team runs on what actually works |
